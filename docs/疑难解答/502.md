# 页面502 Bad Gateway
作者：[小满1221](https://www.yhdzz.cn/)


## 原因

1. PHP没有正常启动或PHP版本选择错误
2. 跨站攻击问题
3. PHP配置出错

## 解决方案

1. 检查PHP版本，确认PHP正常运行
2. 网站设置里关闭跨站攻击后重新打开或者到wp的根目录查到是否有`.user.ini`文件，修改文件中的网站目录到目前的目录（不要删除后面的的:tmp)或者删除此文件
3. PHP配置出错的情况下一般会导致PHP无法正常启动，这种情况在宝塔的情况很多，如果前面编辑过PHP的文件然后后期加了扩展就可能会导致此问题，查看对应PHP的配置文件并修复（一般情况下是两条命令重叠了在一行，建议手动修改的时候多按两次回车避免面板加配置的时候出现两个配置在一行的情况）
4. 查看你的CDN系统，是不是解析错了地址或者解析到了内网地址
